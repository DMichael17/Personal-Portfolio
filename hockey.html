Hockey Registration Relational Database System

This project involved designing and implementing a relational database system to manage hockey league registrations. The goal was to organize player, team, season, and league data into a structured database that reduces redundancy and improves data integrity.

SQL Database Design Normalization Microsoft Access Excel

Problem

Manual tracking of player registrations and team assignments can lead to inconsistent records and duplicated data. The objective of this project was to design a normalized database that allows efficient storage, retrieval, and updating of hockey league registration information.

My Role

  • Designed relational tables for players, teams, leagues, seasons, and registrations
  • Created primary and foreign key relationships
  • Applied normalization principles to reduce redundancy
  • Validated and organized data using Excel
  • Produced documentation and system presentation materials

Implementation

The database was structured using a central registration table connected to lookup tables such as player, team, league, and season. This design allows flexible tracking of player participation across seasons and teams while maintaining data consistency.

Results

  • Improved organization of registration data
  • Reduced manual tracking complexity
  • Demonstrated understanding of relational database design
  • Showed ability to document and present technical systems
